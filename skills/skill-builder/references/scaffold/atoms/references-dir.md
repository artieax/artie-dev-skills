# atom: references-dir

Adds a `references/` directory next to `SKILL.md` for supplementary docs and append-heavy data (logs, eval scores).

## When to include

- The Workflow section in SKILL.md exceeds ~40 lines
- The skill needs an `eval-log.jsonl` for tracking iterations across versions (committed ledger; the optimizer cache `data/evals.jsonl` lives elsewhere — see [`evals/index.md#two-eval-files-ledger-vs-cache`](../../evals/index.md#two-eval-files-ledger-vs-cache))
- Protocol details would clutter the main SKILL.md

If none apply, skip — the directory is optional.

## Layout

```
skills/<name>/
├── SKILL.md
└── references/
    ├── <topic>.md           # one file per major sub-topic
    └── eval-log.jsonl       # optional, committed append-only summary ledger
```

> Sibling — *not* added by this atom: `data/evals.jsonl` is the optimizer training cache regenerated by `scripts/collect_evals.py`. It is gitignored and only appears when the skill ships `scripts-dir`. See [`evals/index.md`](../../evals/index.md#two-eval-files-ledger-vs-cache).

## File conventions

- One file per major sub-topic — `evals.md`, `iteration.md`, `dependency.md`
- Lowercase, dash-separated names ending in `.md` or `.jsonl`
- Keep each reference focused — a reference covering everything is just another cluttered SKILL.md
- `eval-log.jsonl` follows the format defined in `references/evals/index.md` of `skill-builder` (committed ledger only; the optimizer cache is `data/evals.jsonl`)

## SKILL.md integration

Each delegated workflow step links to its reference:

```markdown
### 2. <Step>

Full protocol → `references/<topic>.md`

<2–4 line summary so the step is actionable without opening the reference>
```

## When to upgrade

If the skill grows enough that two or more references could each have an independent trigger phrase, consider the `split` preset (`scaffold/split.md`).
